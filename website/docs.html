Getting Started

Installation

npm install tonl

Quick Start (30 seconds)

import { TONLDocument } from 'tonl';

// Create or load a document
const doc = TONLDocument.fromJSON({
  users: [
    { name: 'Alice', age: 30, role: 'admin' },
    { name: 'Bob', age: 25, role: 'user' }
  ]
});

// Query the data
const admins = doc.query('users[?(@.role == "admin")]');
console.log(admins); // [{ name: 'Alice', ... }]

// Modify the data
doc.set('users[0].age', 31);

// Save to file
await doc.save('data.tonl');

That's it! You're using TONL!

Query, modify, and save your data with a powerful API.

API Reference

TONLDocument Class

The primary interface for working with TONL data.

Factory Methods

  • TONLDocument.parse(tonlText) - Parse TONL string
  • TONLDocument.fromJSON(data) - Create from JavaScript
  • TONLDocument.fromFile(path) - Load from file

Query Methods

  • get(path) - Get value at path
  • query(expression) - Advanced queries
  • exists(path) - Check if path exists
  • typeOf(path) - Get value type

Modification Methods

  • set(path, value) - Create/update
  • delete(path) - Remove
  • push(path, ...items) - Add to array
  • pop(path) - Remove from array
  • merge(path, object) - Merge objects

Query Syntax

Path Expressions

user.name - Property access
users[0] - Array index
users[-1] - Negative index (last)
users[*].name - Wildcard
$..email - Recursive descent
users[0:5] - Array slicing

Filter Expressions

users[?(@.age > 18)] - Comparison
users[?(@.active == true)] - Equality
users[?(@.age > 18 && @.verified)] - Logical AND
users[?(@.email contains "@company.com")] - String operators
files[?(@.name startsWith "test")] - String matching

Operators

Comparison

==, !=, >, <, >=, <=

Logical

&&, ||, !

String

contains, startsWith, endsWith, matches

CLI Tools

Available Commands

ENCODE - Convert JSON to TONL
tonl encode data.json --out data.tonl --smart --stats
DECODE - Convert TONL to JSON
tonl decode data.tonl --out data.json --strict
QUERY - JSONPath queries
tonl query users.tonl 'users[?(@.role == "admin")]'
GET - Simple path retrieval
tonl get config.tonl "database.host"
STATS - Size and token comparison
tonl stats data.json --tokenizer gpt-5
VALIDATE - Schema validation
tonl validate data.tonl --schema data.schema.tonl
